Summary

This plugin creates a navigation bar as seen on Google when you limit articles or search for something. It has a wide variety of attributes so you are able to customize it until you drop.

Attributes

maximumtabs

The maximum number of tabs to display. Can not be set to 1 since that kills the whole idea of this plugin.

  • 11 – default.

linkcurrent

Wheter or not the current page number is a link or not.

  • 0 – No link (default).
  • 1 – Link.

outputlastfirst

Wheter or not to output the First/Last tabs.

  • 1 – Output (default).
  • 0 – Don’t.

outputnextprevious

Wheter or not to output the Next/Previous tabs.

  • 1 – Output (default).
  • 0 – Don’t.

ulid

Enables you to set a id to the <ul> that wraps the output.

ulclass

Enables you to set a class to the <ul> that wraps the output.

liclass

Sets a default class to all <li> tabs.

liselected

Adds this class to the currently active <li> tab.

liselectedtext

If you want to have the selected tab output something else then the number, add it here.

liempty

Adds this class to all <li> tabs that are empty.

firsttext

Enables you to alter the text inside the first link.

firsttexttitle

Enables you to alter the title of the first link.

previoustext

Enables you to alter the text inside the previous link.

previoustexttitle

Enables you to alter the title of the previous link.

nexttext

Enables you to alter the text inside the next link.

nexttexttitle

Enables you to alter the title of the next link.

lasttext

Enables you to alter the text inside the last link.

lasttexttitle

Enables you to alter the title of the last link.

pagetext

Enables you to alter the text in the titles of the page tabs.

reversenumberorder

Makes it possible to reverse the numbers in the tabs.

  • 0 – 1,2 and so on (default).
  • 1 – 2,1 and so on.

moretabsdisplay

Want to get a <li> that states that there are more tabs before or after the ones currently viewable? Make this value to a 1 then.

moretabstext

If you just added the attribute moretabsdisplay to 1, then you need to define where it should show. Want it in the start if there are pages below the current ones? Or do you want it after if there are more tabs available? You don’t need to choose, just comma-separate the below values and you’re done.

  • before
  • after

wraptag

No brainer. Does what it usually does in TXP. Wraps the tag specified around the output.

Examples

<txp:ob1_googlenav firsttext="First" previoustext="Previous" nexttext="Next" lasttext="Last" />

This outputs if there are three pages

  • First
  • Previous
  • 1
  • 2
  • 3
  • Next
  • Last

Do note that this plugin creates an unordered list that is meant to be styled by you, through CSS.

Changelog

Version 2.5 [2007-10-13]

Changed features

  • New name on popular demand: ob1_googlenav became ob1_pagination.
  • Removed outputlinktags feature. In TXP 4.x it’s hard to get it to work. Maybe will release it a separate plugin, who knows?

New features

  • Attribute outputnextprevious added. Does what the name specifies.

Version 2.4 [2007-09-18]

Bug fixes

  • Now works with TXP 4, thanks to Gerhard for adressing the most parts of this. This included:
    • messy_urls didn’t work after TXP RC3 (or something).
    • breaking XHTML since the use of & in the URLs.
  • Attribute outputfirstlast didn’t work properly. Now does what it’s supposed to do.
  • Attribute liselected was not added if not the attribute linkcurrent was chosen. Fixed.

Changed features

  • Attribute maxtab changed to maximumtabs for better naming procedure. Still does the same.
  • liclass is added to all <li>.
  • liempty is added to all empty <li>’s, not just setting it as one and only class.
  • liselected is added to the selected <li> like liempty above.

New features

  • New attributes:
    • liselectedtext
    • reversenumberorder
    • moretabsdisplay
    • moretabstext
    • wraptag

Read up on them above.

Older versions
